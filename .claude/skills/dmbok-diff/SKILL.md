# DMBOK 差分比較（アセスメント比較・推移分析）

## 概要

2つ以上のアセスメント結果（JSON）を比較し、スコアの変化・改善箇所・悪化箇所を可視化するレポートを出力する。

## 実行フロー

### Step 1: 比較対象の特定

`output/` ディレクトリ内のアセスメント JSON を探す:

```bash
ls output/assessment_*.json
```

- ファイルが2つ以上ある場合、ユーザーに比較対象を確認する（デフォルトは最古と最新）
- ファイルが1つしかない場合、「比較対象がありません。先に新しいアセスメントを実行してください」と案内する
- ユーザーが明示的にファイルパスを指定した場合はそれに従う

### Step 2: アセスメント JSON の読み込み

比較対象の2つの JSON を読み込み、以下を確認する:

1. 両方とも `assessment_schema.json` に準拠していること
2. `scores` 配列が11領域分あること
3. `date` フィールドで前後関係を判定する（日付が古い方を「前回」、新しい方を「今回」とする）

### Step 3: 差分分析

11領域それぞれについて以下を分析する:

#### 3a. スコア変化の分類

各領域のスコア変化を以下の3カテゴリに分類する:

- **改善（Improved）**: スコアが上昇した領域
- **悪化（Regressed）**: スコアが下降した領域
- **維持（Unchanged）**: スコアが変わらない領域

#### 3b. 詳細な変化分析

スコアが変化した領域については、前回と今回の以下を比較する:

- **強み（strengths）の変化**: 新たに認められた強み / 失われた強み
- **課題（issues）の変化**: 解消された課題 / 新たに発生した課題
- **改善提案（recommendations）の変化**: 達成された提案 / 新規の提案

この比較は完全一致ではなく、内容の意味的な類似性で判断すること。表現が異なっていても同じ内容を指していれば同一とみなす。

#### 3c. 総合サマリー

以下の観点で総合的なサマリーを作成する:

- 全体平均スコアの変化（前回 → 今回）
- 最も改善が進んだ領域とその要因
- 悪化した領域がある場合、その原因の推察
- 前回のロードマップ（short_term / mid_term / long_term）に対する進捗状況
- 次に注力すべき領域の提案

### Step 4: Markdownレポート出力

`output/diff_YYYYMMDD_vs_YYYYMMDD.md` に比較レポートを出力する。

レポートには以下を含める:

1. 比較概要（対象、前回日付、今回日付、全体スコア変化）
2. スコア変化サマリーテーブル（11領域 × 前回スコア / 今回スコア / 変化）
3. 改善された領域の詳細
4. 悪化した領域の詳細
5. 前回ロードマップの進捗確認
6. 今後の推奨アクション

### Step 5: リッチ HTML レポート生成

差分分析結果を JSON にまとめ、HTML レポートを生成する:

```bash
python .claude/skills/dmbok-diff/scripts/generate_diff_report.py output/assessment_BEFORE.json output/assessment_AFTER.json --out-dir output/
```

PDF も生成する場合:

```bash
python .claude/skills/dmbok-diff/scripts/generate_diff_report.py output/assessment_BEFORE.json output/assessment_AFTER.json --out-dir output/ --pdf
```

生成されるファイル:
- `output/diff_YYYYMMDD_vs_YYYYMMDD.html` — オーバーレイレーダーチャート・変化ハイライト付きリッチレポート
- `output/diff_YYYYMMDD_vs_YYYYMMDD.pdf` — PDF版（`--pdf` 指定時）

## 注意事項

- レポートは日本語で記述する
- スコアの変化だけでなく「なぜ変化したか」の分析を重視する
- 悪化した領域があっても批判的にならず、原因と対策を建設的に提示する
- 評価の制約（データソースの違い、評価者の違い等）がある場合は明記する
- 3つ以上のアセスメントがある場合は、時系列での推移も言及する
