# 01. データガバナンス (Data Governance)

## 概要

データガバナンスは、データマネジメント活動全体に対する権限と統制を行使するための計画立案、監視、実施の仕組みである。他の10領域すべてを横断する「ハブ」として機能し、組織のデータに関する意思決定の枠組みを提供する。

## 評価観点

| # | チェック項目 | 確認内容 |
|---|-------------|----------|
| 1 | データガバナンスポリシーの存在 | データの取り扱いに関するポリシー文書が策定・公開されているか |
| 2 | 役割と責任の定義 | データオーナー、データスチュワードなどの役割が明確に定義されているか |
| 3 | 意思決定プロセス | データに関する意思決定のエスカレーションパスが明確か |
| 4 | データ標準の策定 | 命名規則、データ定義、分類基準などの標準が存在するか |
| 5 | コンプライアンス対応 | 法規制（個人情報保護法等）への準拠体制があるか |
| 6 | 変更管理プロセス | データ構造やフローの変更に対するレビュー・承認プロセスがあるか |
| 7 | データ課題の管理 | データに関する課題・インシデントの管理プロセスがあるか |
| 8 | 教育・啓発活動 | データリテラシー向上のための教育プログラムがあるか |

## 成熟度レベル定義

| レベル | 状態 |
|--------|------|
| 1 - 初期 | ガバナンスの概念がなく、データ管理は完全に属人的。ポリシーや標準は存在しない |
| 2 - 反復可能 | 一部のチームでデータ管理のルールが存在するが、組織横断的な標準はない |
| 3 - 定義済み | ガバナンスポリシーが文書化され、役割・責任が定義されている。標準が存在する |
| 4 - 管理済み | ガバナンス活動が定量的に測定され、定期的にレビューされている |
| 5 - 最適化 | ガバナンスが組織文化に根付き、継続的に改善されている。自動化された監視がある |

## Git/Backlogで確認すべきポイント

### Gitリポジトリ
- `CONTRIBUTING.md` や開発ガイドラインの存在
- コードレビュー（PR テンプレート、レビュープロセス）の仕組み
- データ関連の命名規則の一貫性（テーブル名、カラム名）
- マイグレーションファイルの変更管理履歴

### Backlog
- データガバナンス関連のチケットカテゴリ
- データに関するインシデント管理チケット
- ポリシー・標準に関するWikiページ
- レビュー・承認フローの設定

## よくある課題と改善策

| 課題 | 改善策 |
|------|--------|
| ガバナンスが形骸化している | 定期的なレビュー会議の設定、KPIの設定 |
| 役割が不明確 | RACI表の作成、データオーナーの任命 |
| 標準が守られない | 自動チェック（Linter等）の導入、CIへの組み込み |
| 変更管理が属人的 | PR必須化、データ変更の承認フロー策定 |
