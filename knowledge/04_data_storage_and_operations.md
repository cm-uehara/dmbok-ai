# 04. データストレージとオペレーション (Data Storage & Operations)

## 概要

データストレージとオペレーションは、データの保管場所の設計・実装・管理、およびデータベースの日常的な運用・保守を担う領域である。データの可用性・パフォーマンス・リカバリを確保する。

## 評価観点

| # | チェック項目 | 確認内容 |
|---|-------------|----------|
| 1 | データベース運用手順 | DB運用に関する手順書・ランブックが存在するか |
| 2 | バックアップ戦略 | 定期バックアップの設定・リストアテスト手順があるか |
| 3 | 災害対策（DR） | DR計画が策定され、RTOとRPOが定義されているか |
| 4 | パフォーマンス管理 | DB パフォーマンスの監視・チューニング体制があるか |
| 5 | 容量管理 | ストレージ容量の監視・増設計画があるか |
| 6 | データ保持ポリシー | データの保持期間・アーカイブ・削除ポリシーが定義されているか |
| 7 | 環境管理 | 本番/ステージング/開発環境が適切に分離されているか |
| 8 | デプロイメント | DBスキーマ変更のデプロイ手順が標準化されているか |

## 成熟度レベル定義

| レベル | 状態 |
|--------|------|
| 1 - 初期 | 運用手順がなく、障害対応が場当たり的。バックアップが不十分 |
| 2 - 反復可能 | バックアップは取得しているが、リストアテストが未実施。手順書が一部存在 |
| 3 - 定義済み | 運用手順が文書化され、バックアップ・DR計画が策定されている |
| 4 - 管理済み | パフォーマンスと容量が定量的に監視され、SLAが定義されている |
| 5 - 最適化 | 運用が自動化され、自己修復機能やオートスケーリングが実装されている |

## よくある課題と改善策

| 課題 | 改善策 |
|------|--------|
| バックアップのリストアテスト未実施 | 定期的なリストア訓練の実施、自動テストの構築 |
| パフォーマンス劣化 | スロークエリログの監視、インデックス最適化 |
| 環境差異 | IaC の導入、環境構築の自動化 |
| 容量不足の突発対応 | 容量監視アラートの設定、増設計画の策定 |
