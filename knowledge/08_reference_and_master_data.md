# 08. 参照データとマスターデータ (Reference & Master Data)

## 概要

参照データとマスターデータ管理は、組織横断で共有されるコアデータ（マスターデータ）と分類・コード体系（参照データ）を一元的に管理する領域である。データの一貫性と信頼性の基盤を提供する。

## 評価観点

| # | チェック項目 | 確認内容 |
|---|-------------|----------|
| 1 | マスターデータの定義 | 組織のマスターデータ（顧客、商品、組織等）が明確に定義されているか |
| 2 | 参照データの管理 | コード値、区分値等の参照データが一元管理されているか |
| 3 | シングルソースオブトゥルース | マスターデータの正本（Golden Record）が定義されているか |
| 4 | データ同期 | マスターデータの各システムへの配布・同期方法が定義されているか |
| 5 | マスター変更管理 | マスターデータの変更に対する承認プロセスがあるか |
| 6 | コード体系の標準化 | コード値の命名規則・体系が標準化されているか |
| 7 | 重複排除 | マスターデータの重複検知・排除の仕組みがあるか |

## 成熟度レベル定義

| レベル | 状態 |
|--------|------|
| 1 - 初期 | マスターデータの定義がなく、各システムが独自にデータを持っている |
| 2 - 反復可能 | 主要なマスターデータは認識されているが、一元管理されていない |
| 3 - 定義済み | マスターデータの一覧・定義が文書化され、正本が決められている |
| 4 - 管理済み | マスターデータの品質が定量的に管理され、同期状態が監視されている |
| 5 - 最適化 | MDM基盤が導入され、マスターデータの自動整合・品質管理が実現されている |

## Git/Backlogで確認すべきポイント

### Gitリポジトリ
- マスターデータのシードファイル（`seeds/`, `fixtures/`）
- 定数定義・Enum定義ファイル
- マスター管理用API / 管理画面のコード
- 参照データのCSV / YAMLファイル
- マスターデータのバリデーションロジック

### Backlog
- Documents/Wiki のデータ関連ドキュメント
- マスターデータ変更に関するチケット
- コード体系に関するWiki
- マスターデータ関連のインシデント

## よくある課題と改善策

| 課題 | 改善策 |
|------|--------|
| 同じデータが複数システムに重複 | MDM（Master Data Management）基盤の検討、正本の定義 |
| コード値が統一されていない | コード体系の標準化、マッピングテーブルの整備 |
| マスター変更の影響が不明 | 影響分析プロセスの導入、データリネージの活用 |
| 参照データがハードコード | 設定ファイル・DBへの外出し、管理画面の整備 |
