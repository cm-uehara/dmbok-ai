# 10. メタデータ管理 (Metadata Management)

## 概要

メタデータ管理は、「データに関するデータ」を体系的に収集・保管・統合・活用する領域である。ビジネスメタデータ（用語定義）、テクニカルメタデータ（スキーマ情報）、オペレーショナルメタデータ（処理ログ）を含み、データの発見・理解・信頼性の基盤を提供する。

## 評価観点

| # | チェック項目 | 確認内容 |
|---|-------------|----------|
| 1 | データカタログ | データ資産の一覧（テーブル、カラム、API等）を検索・参照できるカタログがあるか |
| 2 | ビジネス用語集 | ビジネス用語とその定義が管理されているか |
| 3 | テクニカルメタデータ | テーブル定義・カラム説明・データ型等が文書化されているか |
| 4 | データリネージ | データの出自・変換経路が追跡できるか |
| 5 | データオーナーシップ | 各データの責任者が明確にされているか |
| 6 | メタデータの自動収集 | スキーマ情報等のメタデータが自動的に収集されているか |
| 7 | データ辞書 | カラムレベルの詳細な定義・制約・サンプル値を含む辞書があるか |

## 成熟度レベル定義

| レベル | 状態 |
|--------|------|
| 1 - 初期 | メタデータが管理されておらず、データの意味や出自が不明 |
| 2 - 反復可能 | 一部のテーブルにコメントやドキュメントがあるが体系的でない |
| 3 - 定義済み | データカタログまたはデータ辞書が整備され、ビジネス用語集が存在する |
| 4 - 管理済み | メタデータが自動収集され、データリネージが可視化されている |
| 5 - 最適化 | メタデータがデータガバナンスと統合され、データの発見・理解・信頼性評価に活用されている |

## Git/Backlogで確認すべきポイント

### Gitリポジトリ
- DDL内のテーブル・カラムコメント
- データ辞書ファイル（`data_dictionary.md`, `glossary.yml` 等）
- dbt の `schema.yml`（テーブル・カラムの説明）
- API ドキュメント内のデータ定義
- ORM モデルの docstring / コメント
- README 内のデータ構造の説明

### Backlog
- データ定義に関する問い合わせチケット
- 用語集・データ辞書のWikiページ
- データの意味確認に関するやりとり

## よくある課題と改善策

| 課題 | 改善策 |
|------|--------|
| テーブルの意味がわからない | DDLコメントの義務化、データ辞書の整備 |
| データカタログがない | DataHub / Amundsen 等の OSS カタログの導入検討 |
| データの出自が追えない | dbt等でリネージを自動生成、データフロー図の整備 |
| ビジネス用語の認識がずれている | ビジネス用語集の策定、用語定義レビュー会の実施 |
